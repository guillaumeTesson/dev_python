		<h2>Définition</h2>
		<p>Les numéros de port <a href="https://www.frameip.com/entetetcp/">TCP</a> et <a href="https://www.frameip.com/entete-udp/">UDP</a> sont codé sur 16 bits délimitant ainsi leurs valeurs entre 0 et 65535. Les ports connus ont la particularité d'être assignés par l'Iana. Ils sont, dans la plus part des operating system, utilisés par des process en mode root, admin ou encore, avec des doits utilisateurs avancés. Vous retrouverez les détails de fonctionnement des ports avec Tcp et Udp dans la <a href="https://www.frameip.com/rfc-793-transmission-control-protocol/">RFC 793</a> et la <a href="https://www.frameip.com/rfc-768-user-datagram-protocol/">RFC 768</a>. La liste est divisée en trois parties nommées :</p>
		<ul>
			<li class="chapitre---texte-de-niveau-1">Les port connues, compris entre 0 et 1023</li>
			<li class="chapitre---texte-de-niveau-1">Les ports enregistrés, compris entre 1024 et 49151</li>
			<li class="chapitre---texte-de-niveau-1">Les ports dynamique ou privés, compris entre 49152 et 65535.</li>
		</ul>
		<h2>Recherche</h2>
		<p>Voici la possibilité de rechercher un Port parmi la liste complète :</p>

<p>En fait, je pense que tu fais une erreur d&rsquo;interprétation sur la définition du mot « ICMP », mais ne t&rsquo;inquiète pas, tu n&rsquo;es pas le seul 🙂</p>
<p>ICMP est un protocole, cela n&rsquo;a vraiment rien à voir avec une appli de type ping. Donc ICMP est un protocole de niveau 3 permettant de gérer les informations relatives aux erreurs d&rsquo;IP. Et il est vrai, que le champs Type offre le code 0 et 8. C&rsquo;est une fonction d&rsquo;echo réponse.</p>
<p>La commande ping est une application en vue de tester la réponse (donc présence) d&rsquo;un hôte tout en lui mesurant son temps de réponse.</p>
<p>Donc pour réaliser cela, la commande ping sous Windows par exemple, utilise ICMP en code 0,8. mais se n&rsquo;est pas obligatoire, par exemple, la commande ping sous Linux, utilise UDP 7 (echo). Idem pour un Nmap (pas en mode scan, mais en mode test de présence).</p>
<p>Donc, on ne peut pas dire que ping se doit de fonctionner en ICMP. Mais on peux dire que pour mesurer, on a le choix entre ICMP, et port 7.</p>
<p>Tu trouveras tous les détails dans la RFC 862. Lol quand tu l&rsquo;ouvriras 🙂</p>
<p>Après tu fais références  au modèle OSI, repose ta question plus précisement, mais je peux déjà te dire que seul le modèle TCP/IP correspond à TCP/IP et que le modèle OSI n&rsquo;est pa un réel modèle mise en pratique, c&rsquo;est uniquement un modèle de référence (un langage commun à chacun).</p>
<p>N&rsquo;hésite pas Romain, je trouverais toujours du temps pour répondre à tes questions ou remarques.</p>
